## 📌 Problem Statement

Automated AI systems make high-stakes decisions every day — in hiring, lending, healthcare, criminal justice, and beyond. These systems silently **inherit and amplify historical biases**, producing discriminatory outcomes against marginalized groups based on gender, race, age, or socioeconomic status.

Most organizations lack the tools to **detect this bias**, understand its severity, or take corrective action. The result: AI that claims objectivity while perpetuating systemic inequality.

> **FairFrame Pro** solves this by providing a complete end-to-end AI Fairness Auditing platform — from bias detection and visualization, to Gemini-powered reporting and mathematical mitigation.

## 🧠 Technical Architecture

### 4 Core Algorithmic Pillars

---

#### Pillar 1 — Disparate Impact & Statistical Parity Detection

FairFrame computes the industry-standard **EEOC Four-Fifths (80%) Rule**:

```
Disparate Impact (DI)          = Selection Rate (Unprivileged) / Selection Rate (Privileged)
Statistical Parity Difference  = P(Ŷ=1 | unprivileged) − P(Ŷ=1 | privileged)
Fairness Gap                   = |SPD| × 100
```

**Risk Classification (aligned with EEOC & IBM AIF360 standards):**

| Risk Tier | Classification Gap | Regression Ratio | Legal Framing |
|-----------|-------------------|------------------|---------------|
| 🟢 **LOW RISK** | ≤ 5% | ≤ 1.2× | Within acceptable statistical noise |
| 🟡 **MODERATE** | 5% – 15% | 1.2× – 1.5× | Warrants monitoring; possible systemic issue |
| 🔴 **HIGH RISK** | > 15% | > 1.5× | Clear EEOC adverse impact — immediate review required |

---

#### Pillar 2 — Proxy Variable Detection

Many features act as **indirect proxies** for protected attributes (e.g., zip code → race, job title → gender). FairFrame detects these by:

1. Encoding each protected attribute to numeric codes
2. Computing Pearson absolute correlation against all other numeric features
3. Flagging features with `|r| > 0.5` as high-risk proxies

```python
# bias_detector.py — show_proxy_warning()
temp_df[protected_col] = temp_df[protected_col].astype('category').cat.codes
corr = temp_df.corr(numeric_only=True)[protected_col].abs().sort_values(ascending=False)
# Features above threshold are flagged as proxy risks
```

#### Pillar 4 — Kamiran-Calders Reweighing (Bias Mitigation)

A principled pre-processing algorithm that assigns a corrective weight to each training sample based on the joint probability of its group membership and outcome:

```
Weight(group g, outcome y) = [P(group g) × P(outcome y)] / P(group g AND outcome y)
```

This ensures every demographic group achieves equal effective representation during model training — without altering or deleting original data.

```python
# bias_fixer.py — apply_mitigation()
p_gy = len(df[(df['group_key'] == g) & (df['target_bin'] == y)]) / n
weight = (p_group[g] * p_outcome[y]) / (p_gy + 1e-6)
```

## 💻 Using the Generated Fair Dataset

After the Mitigation Engine runs, download `fair_mitigated_data.csv`. Use the auto-generated `weights` column to retrain your model with fairness awareness:

```python
import pandas as pd
from sklearn.ensemble import RandomForestClassifier

# 1. Load the fairness-aware CSV generated by FairFrame
df_new = pd.read_csv("fair_mitigated_data.csv")

# 2. Separate features, target, and the FairFrame-generated weights
X = df_new.drop(['target_column', 'weights', 'protected_attribute'], axis=1)
y = df_new['target_column']
w = df_new['weights']   # ← Bias correction weights from Kamiran-Calders

# 3. Train the model — bias in the protected attribute is now neutralized
model = RandomForestClassifier()
model.fit(X, y, sample_weight=w)

print("✅ Model trained successfully with Fairness-Aware Re-weighting!")
```
